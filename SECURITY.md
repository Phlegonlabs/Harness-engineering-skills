# Security Policy

## Supported Branch

Security fixes are currently supported on:

- `main`

## Reporting a Vulnerability

Do not open a public issue for a security vulnerability.

Report it privately through one of these channels:

1. GitHub Security Advisories for this repository, if enabled
2. Direct private contact with the repository maintainer

Please include:

- a clear description of the issue
- affected repository area or skill package
- reproduction steps
- impact assessment
- any suggested mitigation, if available

## Scope

Reports are especially relevant for:

- unsafe generated scaffold behavior
- hook, guardian, or validation bypasses
- accidental secret exposure
- workflow states that allow agents to skip required checks
- repository changes that can cause unsafe automation outcomes

## Disclosure

Please allow time for triage and remediation before public disclosure.
